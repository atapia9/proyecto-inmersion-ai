Examinar si está Cerrado (XIC)

La instrucción **XIC (eXamine If Closed)** es un contacto que se hace lógicamente verdadero cuando el estado de un bit asociado es **1 (ON)**. Es el equivalente a un contacto normalmente abierto (NO) en un diagrama eléctrico. El flujo de lógica (poder) pasa a través de él cuando el bit está encendido.

Ejemplo en Ladder:

|--[XIC]------(OTE)--|

`XIC` se hará verdadero si su tag es 1. `OTE` se hará verdadero si `XIC` es verdadero.

Examinar si está Abierto (XIO)

La instrucción **XIO (eXamine If Open)** es un contacto que se hace lógicamente verdadero cuando el estado de un bit asociado es **0 (OFF)**. Es el equivalente a un contacto normalmente cerrado (NC) en un diagrama eléctrico. El flujo de lógica (poder) pasa a través de él cuando el bit está apagado.

Ejemplo en Ladder:

|--[XIO]------(OTE)--|

`XIO` se hará verdadero si su tag es 0. `OTE` se hará verdadero si `XIO` es verdadero.

Bobina de Salida Energizada (OTE)

La instrucción **OTE (Output Energize)** es una bobina de salida. Cuando el peldaño que la precede es lógicamente verdadero (el poder fluye hasta ella), la bobina se energiza, lo que significa que el bit asociado se establece en **1 (ON)**. Si el peldaño se vuelve falso, el bit se restablece a **0 (OFF)**.

Ejemplo en Ladder:

|--[XIC Start]--[XIO Stop]--(--OTE Motor--)--|

El `Motor` se energiza si `Start` es verdadero y `Stop` es falso.

Bobina de Salida Latch (OTL)

La instrucción **OTL (Output Latch)** es una bobina que se utiliza para "enganchar" o "latchear" un bit en un estado **1 (ON)**. Una vez que el peldaño se hace verdadero y la bobina se energiza, el bit permanece en 1 incluso si el peldaño se vuelve falso. Para apagar el bit, se debe usar una instrucción OTU con el mismo tag.

Ejemplo en Ladder:

|--[XIC Set_Bit]--(--OTL Bit--)--|
|--[XIC Reset_Bit]--(--OTU Bit--)--|

`OTL` enciende `Bit` y lo mantiene encendido. `OTU` debe usarse para apagarlo.

Bobina de Salida Unlatch (OTU)

La instrucción **OTU (Output Unlatch)** es una bobina que se utiliza para "desenganchar" o "deslachear" un bit, es decir, forzarlo a un estado **0 (OFF)**. Se usa en combinación con la instrucción OTL para crear una lógica de "set-reset". OTU solo tiene efecto cuando su peldaño es verdadero; de lo contrario, no hace nada.

Ejemplo en Ladder:

|--[XIC Set_Bit]--(--OTL Bit--)--|
|--[XIC Reset_Bit]--(--OTU Bit--)--|

`OTU` apaga `Bit` y lo mantiene apagado. `OTL` debe usarse para encenderlo de nuevo.
