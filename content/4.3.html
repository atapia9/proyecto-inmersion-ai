Instrucción EQU (EQUAL)

La instrucción **EQUAL** compara dos valores. Se hace lógicamente verdadera cuando el `Source A` es exactamente igual al `Source B`. Se utiliza comúnmente para activar una lógica cuando un contador o temporizador alcanza un valor específico.

Ejemplo en Ladder:

|--[EQU SourceA:Counter.ACC SourceB:10]--(--OTE Light--)--|

`Light` se enciende cuando el contador llega a 10.

Instrucción LES (LESS THAN)

La instrucción **LESS THAN** compara dos valores. Se hace lógicamente verdadera cuando el `Source A` es menor que el `Source B`. Es útil para mantener una lógica activa mientras un valor se mantiene por debajo de un umbral.

Ejemplo en Ladder:

|--[LES SourceA:Temp.ACC SourceB:500]--(--OTE Heater_On--)--|

`Heater_On` se enciende si la temperatura es menor a 500 grados.

Instrucción GRT (GREATER THAN)

La instrucción **GREATER THAN** compara dos valores. Se hace lógicamente verdadera cuando el `Source A` es mayor que el `Source B`. Se utiliza para activar una alarma o una acción cuando un valor excede un límite de seguridad o de operación.

Ejemplo en Ladder:

|--[GRT SourceA:Pressure_Sensor SourceB:100]--(--OTE Alarm_High--)--|

`Alarm_High` se enciende si la presión del sensor es mayor a 100 psi.

Instrucción OSR (One Shot Rising)

La instrucción **One Shot Rising** se utiliza para detectar una transición de estado de falso a verdadero (o flanco ascendente) en su entrada. Cuando esta transición ocurre, la instrucción se vuelve verdadera por un solo ciclo de escaneo del programa. Esto es crucial para realizar acciones que solo deben ejecutarse una vez por evento, como el reinicio de un contador o la activación de un pulso de un solo disparo.

Ejemplo en Ladder:

|--[XIC Start_Button]--[OSR Reset_OneShot]--(--RES Counter_A--)--|

El contador `Counter_A` se reinicia una sola vez cada vez que se presiona `Start_Button`.
