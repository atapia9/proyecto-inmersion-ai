Indicadores de Estado (LEDs)

Los controladores Allen-Bradley tienen LEDs que proporcionan información vital sobre su estado de funcionamiento y posibles fallas. Entender estos indicadores es el primer paso para el diagnóstico.

OK LED (Verde): Muestra el estado operativo del controlador. Si está fijo, el controlador está en modo RUN y todo está bien. Si parpadea, puede estar en modo de programación o en un estado de espera. Si está en rojo, indica una falla mayor.

I/O LED (Amarillo): Parpadea para indicar que el controlador está en comunicación con los módulos de entrada y salida.

FAULT LED (Rojo): Un LED rojo fijo o parpadeante indica una falla. Esto puede ser una falla de hardware (por ejemplo, un módulo defectuoso) o una falla de software (por ejemplo, un error en el programa).

Proceso Lógico para la Solución de Problemas

Un enfoque estructurado es crucial para diagnosticar y resolver fallas de manera eficiente. Sigue estos pasos:

1. Observar y Analizar: Comienza por verificar los LEDs del PLC y los módulos, y revisa la pantalla de fallas en Studio 5000. Los códigos de falla te darán una pista sobre la causa del problema.

2. Localizar el Problema: Utiliza las herramientas de Studio 5000 como el modo en línea (Online Mode) para monitorear el programa en tiempo real. Observa el estado de los tags de entrada y salida y el flujo de la lógica para identificar dónde se detiene el proceso.

3. Hipótesis y Pruebas: Formula una hipótesis sobre la causa (por ejemplo, "el sensor no está funcionando", "hay un error de lógica") y prueba tu teoría. Puedes forzar bits, inspeccionar valores de tags y usar el modo de un solo paso para verificar la lógica.

4. Implementar una Solución: Una vez que hayas identificado la causa raíz, implementa la solución. Esto puede ser reemplazar un componente, corregir un error de cableado o modificar el código del programa.

Problemas Comunes

Algunos de los problemas más frecuentes en los sistemas de PLC y cómo abordarlos:

Fallas de E/S:

Asegúrate de que el módulo esté bien asentado, que el cableado sea correcto y que el dispositivo de campo (sensor, botón) esté funcionando. Monitorea el estado del bit de entrada en línea para ver si cambia de estado cuando se activa el dispositivo físico.

Fallas de Lógica de Programa:

Usa el modo en línea para monitorear el flujo de la lógica. Si una bobina de salida (OTE) no se energiza, revisa los contactos (XIC, XIO) que la preceden para ver cuál no está cumpliendo su condición. Revisa si hay tags no inicializados, bucles infinitos o valores que excedan los límites de datos.
